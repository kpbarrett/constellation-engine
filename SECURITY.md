# Security Policy

## Reporting a Vulnerability

Do not open a public GitHub issue for a suspected security vulnerability that could expose users, operators, contributor credentials, or private infrastructure details.

Instead:

1. contact the maintainers privately
2. provide a concise description of the issue, impact, and reproduction details
3. include any recommended containment steps if immediate action is needed

## Response Expectations

- acknowledge receipt as soon as practical
- assess severity and containment needs
- coordinate a fix or mitigation
- publish a follow-up disclosure when it is safe to do so

## Scope

Security concerns may include:

- authentication or identity boundary failures
- provenance spoofing
- policy bypasses with safety implications
- CI/CD credential exposure
- workflow or extension boundaries that permit privilege escalation
